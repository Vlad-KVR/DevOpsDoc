Flosum DevOps Documentation Help

APIs

All requests to AccelQ from the Flosum come exclusively from Apex.

Name

Url

Description

Test Connection

https://{URL}/awb/api/1.0/{TENANT_CODE}/jobs/{JOB_ID}/validate-ci-job

We use this endpoint to test the connection. The fact is that AccelQ does not have a special endpoint for connection testing, so we make a job validation request with id ‘1’ if we get a response with status success or the error 'Test Job: 1 is not a Continuous Integration job.' so everything is fine, otherwise we will display an error that Accelq will send us (for example invalid username)

Trigger Job

https://{URL}/awb/api/1.0/{TENANT_CODE}/jobs/{JOB_ID}/trigger-ci-job

We use this endpoint to run some kind of AccelQ test.

Get Job Result

https://{URL}/awb/api/1.0/{TENANT_CODE}/runs/{JOB_ID}

We use this endpoint for two purposes. 1. To check if the test has been completed. 2. To get the test results.
